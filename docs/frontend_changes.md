# 前端加密服务修改摘要

我们对`CryptoService`进行了以下关键修改，以提高安全性和可靠性：

## 1. 盐值格式统一

- 修改了`createAndSetServerSalt`方法，将随机盐值生成从Base64格式更改为32位十六进制字符
- 确保前端和后端使用相同的盐值格式

## 2. 盐值获取优先级调整

- 修改了`getEncryptionSalt`方法，使其优先从服务器获取盐值
- 只有在服务器请求失败或返回404时才使用本地缓存的盐值
- 调整后的优先级顺序：
  1. 服务器的盐值
  2. 本地缓存的盐值（仅当服务器不可用时）
  3. 基于用户名的备用盐值（仅当以上都失败时）

## 3. 改进加密私钥Blob处理

- 修改`encryptPrivateKeyWithKey`方法，增加salt参数以确保使用正确的盐值
- 确保盐值和密文都被正确存储在Blob中
- 确保盐值来源清晰可追溯

## 4. 添加会话管理相关方法

- 修改`clearKeyPair`方法，同时清除加密密钥，避免使用旧密钥
- 添加`refreshEncryptionKey`方法，用于登录成功后刷新加密密钥

## 5. 调试日志优化

- 优化调试日志，确保只在开发环境或明确启用调试时输出详细日志
- 使用`process.env.NODE_ENV !== 'production'`条件限制关键日志输出

## 6. 新增的安全特性

- 更一致的盐值处理，确保跨设备兼容性
- 明确的错误处理和降级策略
- 更好的加密元数据管理，便于跨浏览器密钥恢复

## 相关文件

- 修改了`frontend/src/utils/crypto.ts`，实现了上述大部分改进
- 修改了`frontend/src/api/auth.ts`，确保在注册和登录时使用正确的参数调用加密方法
- 创建了`docs/backend_changes.md`，描述了后端需要进行的相应修改
