# 证书验证说明

本项目实现了针对生产环境的服务器证书验证机制，确保用户数据只发送给可信的服务器。

## 证书验证机制

- 在生产环境中，前端会验证服务器的域名和协议，确保只与预期的服务器通信
- 硬编码了服务器证书指纹和公钥哈希，用于验证服务器身份
- 在发送用户凭证前会自动验证服务器身份

## 生产环境构建

要生成验证服务器证书的生产版本：

```bash
# 使用默认证书指纹和公钥哈希构建
npm run build:prod

# 或指定证书指纹和公钥哈希
./scripts/build-production.sh "证书指纹" "公钥哈希"
```

构建后的文件位于 `dist` 目录，可部署到静态服务器。

## 证书更新

当服务器证书更新时，需要更新证书指纹和公钥哈希，重新构建前端应用：

1. 获取新证书的指纹和公钥哈希
2. 更新 `utils/certificateValidator.ts` 中的硬编码值或使用构建脚本传入新值
3. 重新构建应用并部署
